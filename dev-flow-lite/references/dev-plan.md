# 增量改动提案（简单任务专用）
## 文档说明
- 归属体系：Dev-Flow-Lite
- 关联主文档：SKILL.md
- 适用范围：Bug 修复、样式调整、参数修改、单文件微调、文案/注释修正等简单任务
- 使用要求：严格按照主文档规则及下方流程执行，填写完成后提交对应负责人确认。

## 一、任务执行流程
整体流程链路：**输出改动提案 → 负责人确认 → 落地代码修改 → 自查校验 → 统一归档**

1. **输出改动提案**
完整填写本文档内的标准提案模板，由开发人员负责。

2. **等待确认**
输出书面确认或工单批注，由技术负责人/需求对接人负责。

3. **落地修改**
产出改动后代码并补充关键注释，由开发人员负责。

4. **自查校验**
留存自检结果记录，由开发人员负责。

5. **迭代归档**
整理提案原文与 Git 版本记录，由开发人员负责。

## 二、标准模板
```
【增量改动提案】
1. 改动目标
2. 涉及文件
3. 执行步骤
4. 本次不改动范围
5. 兼容风险与注意事项
```

## 三、字段逐项填写说明
1. **改动目标**
填写对应 Bug 单号/需求单号 + 具体问题/优化目标，描述清晰、无歧义。
示例：`BUG-20260612：修复列表页按钮点击无响应问题`

2. **涉及文件**
精确填写**文件完整路径 + 大致代码行号**，禁止模糊描述。
示例：`src/views/list/index.vue 第 120 ~ 135 行`

3. **执行步骤**
分步骤描述修改动作，做到可复现、可核对。

4. **本次不改动范围**
明确划清边界，列出**绝对不修改**的文件、逻辑、配置，防止过度开发。

5. **兼容风险与注意事项**
说明是否影响线上现有功能、是否需要回归测试、有无上下游依赖项。

## 四、完整填写示例
```
【增量改动提案】
1.改动目标：BUG-20260612：修复列表页查询按钮点击无响应问题
2.涉及文件：
- src/views/list/index.vue 第 120 ~ 135 行
3.执行步骤：
- 定位查询按钮绑定的 click 事件；
- 补充缺失的请求参数校验逻辑；
- 本地点击测试，验证按钮正常触发查询。
4.本次不改动范围：列表渲染逻辑、分页组件、样式文件、其他页面代码。
5.兼容风险与注意事项：仅修复前端交互，不影响接口与数据；需回归列表查询功能。
```

## 五、使用约束
1. 本文档仅用于 `SKILL.md` 定义的**简单任务**，复杂功能禁止使用本模板；
2. 所有字段必须如实填写，不得留空、简写敷衍；
3. 提案确认通过后，必须严格按照内容执行修改，不得擅自扩大范围；
4. 改动完成后，提案文档随代码版本、工单统一归档留存。
